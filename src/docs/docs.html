Vix Documentation

Modern. Safe. Performant. The complete guide to the Vix programming language.

Introduction

Vix is a modern programming language designed to balance performance, memory safety, and developer productivity. It combines the speed of C with the safety features of modern languages, making it suitable for both low-level system programming and high-level application development.

Current Status: Alpha version. Expect breaking changes as we rapidly iterate towards stability.

Key Features

  • Performance: Compiles to C, then to LLVM using Clang. Achieves C-level performance with SSA optimizations.
  • Safety: Immutable variables by default, no null (uses None), built-in Option and Result types.
  • Developer Experience: Simple syntax using end keyword instead of braces. Beginner-friendly design.
  • Interoperability: Seamless integration with C/C++ libraries.

Language History

Vix was created by the Vix Development Team under the leadership of MrBatata. The project began in early 2024 and was officially released to the public on January 1, 2026.

Memory Safety

Vix is memory-safe by default. It uses compile-time checks and an ownership system to prevent common memory issues while keeping code readable and efficient. Key points:

  • Automatic memory management by default.
  • Ownership rules prevent dangling pointers and double frees.
  • Mutable and immutable variables control what can be changed.
  • Borrowing via ref / ref mut allows safe access without copying.
  • Move, Copy, and Clone semantics ensure predictable memory usage.

Safe vs Unsafe Blocks

Vix distinguishes between safe and unsafe code:

  • Safe (default):
    • Automatic memory management.
    • Ownership enforced at compile-time.
    • Overflow and boundary checks enabled.
    • Prevents common memory bugs automatically.
  • Unsafe:
    • Allows raw pointers and manual memory management.
    • No overflow or boundary checks.
    • Used only for performance-critical code or system/OS programming.
    • Programmer must ensure safety manually.
// Unsafe block example unsafe: mut i = 0 forever do i += 10 end // Raw pointer operations allowed mut x = ptr* uint8 free(x) end

Option and Result Types

Vix provides Option and Result types to handle potential absence of values or operations that may fail safely. Key points:

  • Option: Represents a value that may or may not exist.
    • Use Some(value) for a value.
    • Use None when no value exists.
  • Result: Represents an operation that can succeed (Ok) or fail (Err).
    • Helps avoid runtime exceptions.
    • Encourages explicit handling of errors.
// Option type i: Option[int32] = None x: Option[int32] = Some(30) // Result type func example(): Result[Option[int32], str] return Ok((100, "Hello world")) end

Using these features together, Vix ensures memory safety while giving the programmer control over performance when needed.

Borrow & Reference

Vix is a memory-safe programming language. It provides multiple safety mechanisms to ensure programs are fast, secure, and free from common issues such as crashes, memory leaks, and security vulnerabilities.

One of the core safety features in Vix is the borrow system, which allows controlled access to variables without transferring ownership.

There are two types of borrows:

Type Description
Immutable Borrow Read-only access to a variable; modification is not allowed
Mutable Borrow Allows both reading and modifying the borrowed variable

Immutable Borrow: Allows you to read a variable without copying it, while guaranteeing that the value cannot be modified through the borrow.

// Immutable borrow example: create i = 10 create x = &i // borrow "i" print(i) // prints 10 print(x) // prints 10 (x reads the same value as i)

Mutable Borrow: Allows both reading and modifying a variable through the borrowed reference.

// Mutable borrow example: create mut i = 10 create mut x = &i // mutable borrow x += 10 print(x) // prints 20 print(i) // prints 20 (original value is modified)
Borrow Safety:
Immutable and mutable borrows prevent hidden or unexpected changes to data. This makes programs easier to debug and reason about, reduces memory usage, and allows the CVix compiler to apply stronger optimizations. Borrowing also helps prevent issues such as race conditions and ensures that multiple threads can safely read shared data.

Borrow vs Reference

Reference: In Vix, the ref keyword is used to create a reference to an existing variable instead of copying its value. A reference acts as another name for the same data in memory.

  • References provide direct access to the original value.
  • No memory copy is performed.
  • References can be immutable or mutable.

Immutable references allow readonly access, while mutable references allow modifying the original variable. This design provides strong safety guarantees while maintaining C level performance. Example usage:

create mut i = 10 create i = 10 ref x = i // immutable reference print(x) // prints 10 x += 5 // error: cannot modify through immutable reference

Mutable references (ref mut) allow you to read and modify the original variable directly without copying its value. This provides both safety and high performance, as changes through the reference affect the original variable in a controlled way. Example usage:

create mut i = 10 ref mut x = i // mutable reference x += 5 print(x) // 15 print(i) // 15

• Immutable references allow multiple readers

• Mutable references allow a single writer

• No copying occurs when using ref

• References are always valid and memory safe

Move & Copy

Ownership in Vix defines which variable is responsible for a piece of data in memory. Every value has a single owner at a time, and when the owner goes out of scope, the value is automatically cleaned up. Ownership ensures memory safety, prevents dangling references, and avoids accidental data corruption. Combined with mutable and immutable references, Vix allows safe, efficient access to data without copying unnecessarily.

Move

Move: Transfers ownership of a variable to another variable. After a move, the original variable becomes invalid. Move prevents double-free and use-after-free bugs, is automatically applied to complex types like strings, arrays, and structs, and allows efficient passing of large data without copying. Key points:

  • Ownership is transferred to the new variable
  • Original variable cannot be used after move
  • Automatic for complex types
  • Ensures memory safety and efficiency

// Move example (strings / complex types) create mut s = "hello" create t = s // ownership moves to t print(t) // prints "hello" print(s) // error: s has been moved and is no longer valid

copy

Copy: Duplicates the value so both the original and new variable remain valid. Copy is automatic for simple types like integers, booleans, and small fixed size values. It is safe, predictable, and does not require explicit cloning for lightweight data. Key points:

  • Original variable remains usable
  • Automatic for simple, stack only types
  • No ownership conflicts occur
  • Ideal for small, cheap to copy data

// Copy example (numbers are Copy types by default) create a = 10 create b = a // copied, a is still valid print(a) // prints 10 print(b) // prints 10 // Changing b does not affect a b += 5 print(a) // still 10 print(b) // 15

Clone

Clone: Creates an explicit copy of a variable, allowing both the original and the new variable to remain valid. Clone is used for non-copyable or complex types like strings, arrays, or structs when you need two independent owners of the same data. Unlike Move, cloning duplicates the data instead of transferring ownership. Key points:

  • Original and cloned variables are independent
  • Useful for heap-allocated or complex types
  • Prevents ownership conflicts while allowing multiple owners
  • Requires explicit use of clone
Example usage:
create mut s = "world" create t = s.clone() // t is a separate copy print(s) // prints "world" print(t) // prints "world" // Modifying t does not affect s t += "!" print(s) // still "world" print(t) // "world!"

Scope

Scope Blocks

In Vix, scope defines where a variable or reference is valid. Variables and references are only accessible within the scope they are created. When a scope ends, variables go out of scope and are automatically cleaned up.

  • Variables are valid from the point of declaration to the end of the block (function, loop, or code block).
  • Immutable and mutable variables follow the same scope rules.
  • References (ref / ref mut) cannot outlive the variable they point to.
  • Nested scopes can have variables with the same name as outer scopes, creating shadowing.
// Example of scope: func example() create mut x = 10 scope create x = 20 // shadows outer x print(x) // 20 end print(x) // 10 (outer x) end

Scopes with Variables and References:

In Vix, scopes define where variables and references are valid. You can create variables, mutable variables, and references within a scope. Variables inside a scope are destroyed when the scope ends. References must always point to a variable that is still in scope.

  • Immutable variables: readonly inside the scope.
  • Mutable variables: can be modified inside the scope.
  • Immutable references: can read the variable but cannot modify it.
  • Mutable references: can read and modify the variable.
  • Shadowing is allowed: inner scopes can declare variables with the same name as outer scopes.
// Scope with mutable variable: scope mut i: i = 10 i += 5 print(i) // 15 end // Scope with immutable variable: scope i: i = 20 // i += 5 // error: i is immutable print(i) // 20 end // Scope with immutable reference: create x = 30 scope ref r: r = ref x print(r) // 30 // r += 10 // error: cannot modify through immutable reference end // Scope with mutable reference: create mut y = 40 scope ref mut r: r = ref mut y r += 10 print(r) // 50 print(y) // 50 (original variable modified) end

Type System:

Vix has a strongly typed system with built-in primitive types for numbers, strings, and booleans. Each type has a default size, but some types can vary depending on architecture or context.

Overview of Vix primitive types:

  • int — Represents integers. Default is int64 (or int32 on 32-bit systems).
  • float — Represents floating-point numbers. Default is float16.
  • str — Represents strings. Internally stored as usize (pointer-sized type).
  • usize — Unsigned pointer-sized integer. Default is int64 or int32 depending on architecture.
  • isize — Signed pointer-sized integer. Default is int64 or int32 depending on architecture.
  • bool — Represents true/false values. In Vix, this type is called maybe.
// Integers create a = 42 // int64 by default create b: int32 = 10 // explicitly 32-bit integer // Floats create f = 3.14 // float16 by default create g: float32 = 2.71 // Strings create name = "Vix" // stored as usize pointer internally // Pointer-sized integers create index: usize = 100 create offset: isize = -50 // Booleans create alive: maybe = true create ready: maybe = false

Type System & Data Structures

Vix supports various built-in types and data structures including arrays, tuples, structs, enums, and hash maps. You can also define methods on structs using impl and create optional constructors like new.

Arrays

  • Fixed-size or dynamic collections of elements of the same type.
  • Access elements by index: array[0].
  • Mutable arrays can be modified after creation.
mut nums: Array[int] = [1, 2, 3, 4, 5] mut names: Array[str] = ["Alice", "Bob", "Charlie"] print(nums[0]) // 1 print(names[1]) // "Bob" // Modify a mutable array nums[2] = 10 print(nums) // [1, 2, 10, 4, 5]

Tuples

  • Immutable ordered collections of heterogeneous elements.
  • Access elements with dot notation: tuple.0, tuple.1.
  • Useful for returning multiple values from functions.
point: (int, int) = (10, 20) info: (str, int) = ("Alice", 30) print(point.0) // 10 print(info.1) // 30

Structs

  • Custom types with named fields.
  • Methods can be added via impl blocks.
  • Fields can be mutable or immutable depending on declaration.
  • Optional constructors like new can be defined for convenience.
struct Test: i: int x: str end impl Test: func new(): Test Test( i = 0, x = "" ) end func test(&mut self, input: int): Result[int, str] if input != 0 then self.i = 10 return Ok(0) end Err("Failed: input is zero") end end

Using Structs & Methods

func main() create i: int = 10 create test = Test create x = test.test(i) end

Enums

  • Enumerations that can hold associated data.
  • Useful for creating variant types with different data in each variant.
enum Stait: Name(str) Health(int32) end

HashMaps

  • Key-value stores for fast lookup. Keys are unique.
  • Can be declared with inline initialization or empty and populated later.
  • Supports str, int, or other types as keys and values.
// Inline initialization create scores = { ["Banana"] = 100 } // Inline with type annotation create i: HashMap[int32, str] = { [1] = "Hello world" [2] = "Banana" } // Create empty HashMap and populate mut scores: HashMap[str, int] = HashMap() scores["Alice"] = 100 scores["Bob"] = 85 print(scores["Alice"]) // 100 print(scores["Bob"]) // 85

Example Functions

  • random() – Returns a pseudo-random number.
  • generate(len) – Generates a random string of length len.
  • These functions can be used with arrays, strings, and other data structures.
func random(): uint64 mut seed: int32 = 123456789 seed = seed * 6364136223846793005 + 1 return seed end func generate(len: int): str mut alphabets: str = "abcdefghijklmnopqrstuvwxyz" mut resulted = "" mut i: int = 0 while i < len do mut index: int = random() % 26 resulted = resulted + alphabets[index] i = i + 1 end return resulted end

Notes: - The new function in impl is optional; structs can be instantiated directly. - Methods on structs can use &self (immutable) or &mut self (mutable). - Enums and HashMaps provide flexible ways to structure and access data safely. - HashMaps can be created empty, with inline initializers, or populated after creation using square brackets. - Arrays and tuples are immutable by default unless declared mut.

Operations & Control Flow

Vix provides standard control flow operations similar to other languages, but with clean syntax using end instead of braces. Blocks are closed with end, and indentation is optional but recommended for readability.

If / Else / Elif

  • Conditional execution based on boolean expressions.
  • if checks a condition, elif provides additional checks, else handles all remaining cases.
create x: int = 10 if x > 20 then print("x is greater than 20") elif x == 10 then print("x is exactly 10") else print("x is something else") end

While Loops

  • Repeated execution while a condition is true.
  • Use break to exit early and continue to skip to the next iteration.
mut i: int = 0 while i < 5 do print(i) i += 1 end // Example with break / continue mut j: int = 0 while j < 10 do j += 1 if j == 5 then continue // skip printing 5 end if j == 8 then break // exit loop early end print(j) end

For Loops

  • Iterates over a range or collection.
  • Supports arrays, hash maps, or numeric ranges.
// Numeric range for i in 0..5 do print(i) end // Iterating over an array mut names: Array[str] = ["Alice", "Bob", "Charlie"] for name in names do print(name) end // Iterating over a HashMap mut scores: HashMap[str, int] = { ["Alice"] = 100 ["Bob"] = 85 } for key, value in scores do print(key + ": " + value) end

Forever Loop

  • Infinite loop until manually exited with break.
mut counter: int = 0 forever do counter += 1 print(counter) if counter == 5 then break end end

Summary / Notes

  • All control flow blocks are closed with end instead of braces.
  • Conditions can use comparison operators: ==, !=, >, <, >=, <=.
  • Logical operators: and, or, not.
  • Use break and continue inside loops for more control.
  • Vix supports iteration over ranges, arrays, and hash maps seamlessly.

Structs

A struct is a way to group related data together under one name.

Basic Syntax

struct Book: mut title: str mut author: str mut pages: int end # Create an instance mut my_book = Book( title = "The Hobbit", author = "J.R.R. Tolkien", pages = 310 ) # Access fields print(my_book.title) my_book.pages = 360 # Modifiable because 'mut' is used

Impl Blocks (Methods)

You can add functions to structs using impl.

struct Player: mut name: str mut health: int mut score: int end impl Player: # Constructor Player(name: str, health: int, score: int) self.name = name self.health = health self.score = score end func take_damage(&mut self, damage: int) self.health -= damage end func is_alive(&self): bool return self.health > 0 end end
Structs vs Enums: Use Structs when all fields should exist at the same time (AND logic). Use Enums when a value can be only one of several options (OR logic).

Functions

Functions are reusable blocks of code identified by a name.

Basic Syntax

func add(a: int32, b: int32): int32 return a + b end

Parameter Ownership

Vix allows fine-grained control over how data is passed to functions.

  • Owned (Default): Function takes ownership. Original variable invalidated.
  • mut: Function takes mutable ownership.
  • ref: Immutable reference (borrowing).
  • mut ref: Mutable reference. Changes persist.
# Mutable Reference Example func increment(input: mut ref int32) input += 1 end mut value = 10 increment(value) # value is now 11 print(value)

Public vs Private

Functions are private by default. Use public to expose them to other files.

# main.vix public func helper(): int32 return 100 end # other.vix import helper from "src/main.vix" print(helper()) # Works

Enums

An enum represents a value that can be one of several options, but only one at a time.

Simple Enums

enum Color: Red, Green, Blue, end favorite = Color.Red

Enums with Data

Variants can hold specific data relevant to that variant.

enum HttpResponse: Success(data: str, status_code: int), NotFound(url: str), ServerError(error_message: str), end response = HttpResponse.Success(data: "Hello", status_code: 200)

Pattern Matching

Use match to handle enum variants safely.

match response do case HttpResponse.Success(data, code) do print("Success: ", data) end case HttpResponse.NotFound(url) do print("Could not find: ", url) end case HttpResponse.ServerError(msg) do print("Server Error: ", msg) end end

Compiler

The Vix compiler architecture is designed for optimization and safety.

Compilation Pipeline

Source Code (.vx) ↓ Vix Lexer/Parser ↓ Type Checking & AST ↓ Vix Code Generator ↓ C Intermediate Representation ↓ Clang (LLVM) -O3 ↓ Binary

The compiler is written in Rust and utilizes Static Single Assignment (SSA) for optimization. It generates highly optimized C code which is then compiled by Clang.

Performance

Vix achieves C-level performance through Clang optimizations.

Optimization Levels

  • Default: -O3 (Maximum optimization)
  • Inline function expansion
  • Loop unrolling
  • Dead code elimination

Unsafe for Speed

Warning: For maximum performance, use unsafe blocks. This removes bounds checking and borrow checking overhead.
